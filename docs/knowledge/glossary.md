# Cricket Glossary (Quick)

Short, retrieval-friendly definitions. See linked docs for details.

## Formats

- **Test**: multi-day cricket. No over limit per innings. Patience + endurance.
- **ODI**: 50 overs per side. Balances accumulation with controlled aggression.
- **T20**: 20 overs per side. High intent; phase-based strategy is important.

See: `formats/t20_vs_odi_vs_test.md`

## Match phases (T20)

- **Powerplay (PP)**: early overs with field restrictions; scoring opportunities + wicket risk.
- **Middle overs**: consolidation + matchup targeting; pace of the innings is managed here.
- **Death overs**: final overs; high run rates; yorkers/variations; higher boundary rate.

See: `tactics/batting_phases.md`, `tactics/bowling_death_overs.md`, `rules/powerplay.md`

## Batting roles

- **Anchor**: stabilizes innings; scores at a controlled rate; reduces collapse risk.
- **Finisher**: attacks late; optimizes boundary-hitting in death overs.
- **Accumulator**: rotates strike consistently; keeps run rate ticking.
- **Floater**: promoted/demoted based on matchup or situation (e.g., vs spin).

## Bowling roles

- **Powerplay bowler**: targets early wickets; manages swing/seam; protects boundary.
- **Middle-overs bowler**: controls run rate; builds pressure; creates wicket chances via dots.
- **Death bowler**: bowls at the end; uses yorkers, wide lines, slower balls; defends high targets.

## Common terms

- **Economy rate**: runs conceded per over (based on *legal balls*). See `stats/economy.md`.
- **Strike rate (batting)**: runs scored per 100 balls faced (legal balls faced). See `stats/strike_rate.md`.
- **Dot ball**: a ball with 0 runs scored (definition can vary: total runs vs off-the-bat). See `stats/common_metrics.md`.
- **Boundary**: usually a 4 or 6 *off the bat* (some analyses include byes/leg-byes separately). See `stats/common_metrics.md`.
- **DLS**: Duckworth–Lewis–Stern method for rain-affected limited-overs matches. See `rules/dls.md`.
- **Super over**: tie-breaker in limited-overs matches. See `rules/super_over.md`.
